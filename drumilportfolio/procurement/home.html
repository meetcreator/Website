✦ Source to pay

How a PO flows through
the system

Six stages. Fully automated where possible. Human-reviewed where it matters.

Upload any PO file — zero manual mapping

Drag and drop a CSV exported from Tally, or an Excel sheet from any ERP. The ingestion engine auto-detects aliases like "Voucher Number", "Party Name", "WhatsApp No" and more.

POs are grouped by PO number, line items are attached, and the vendor is automatically upserted into the vendor directory.

CSV / XLS / XLSX Auto column detection Line item grouping Vendor upsert POST /ingestion/excel

Vendor gets a WhatsApp confirmation instantly

A dispatch_whatsapp_outbox job is committed to the database and processed by the worker. The vendor receives a confirmation request asking them to acknowledge the PO and share their ETA.

The workflow transitions from CREATED → VENDOR_PENDING automatically.

Meta Cloud API Durable job queue VENDOR_PENDING state Multilingual messages

Vendor replies → HMAC-verified, idempotent webhook

When the vendor replies on WhatsApp, Meta sends a webhook to POST /webhooks/whatsapp. The handler verifies the X-Hub-Signature-256 signature and checks the message_id against the idempotency ledger to prevent duplicate processing.

HMAC-SHA256 verified Idempotency deduplication Text / button / interactive Correlation ID tracing

AI extracts intent, ETA, language, and delay flags

The message body is sent to GPT-4o-mini with a structured prompt. The model returns intent (ack_po, provide_eta, delay_notice, price_discrepancy), an ISO ETA date, a delay flag, a reason, and the dominant language (en/gu/hi).

If no OpenAI key is set, a deterministic rule-based fallback handles Gujarati, Hindi, and English keywords.

GPT-4o-mini JSON structured output Rule-based fallback Gujarati / Hindi / English

High-risk replies are staged for human review

The HITL router evaluates the parsed intent against routing rules. Delay notices, price disputes, and ETAs exceeding a 3-day variance threshold are staged as editable draft replies — not auto-sent.

An AI-generated draft reply in the vendor's language is pre-filled. The coordinator edits if needed, then approves.

Delay → HITL draft Price dispute → HITL draft ETA variance > 3d → HITL ACK → auto-handled

Human coordinator approves and dispatches

In the Operations Console, the coordinator sees the vendor's original message, the pre-filled AI reply, and a reason for review. They can edit the text inline, then click Approve & Send.

Approval updates the draft, message, and approval records, transitions the workflow to APPROVED, and enqueues the final dispatch job.

Inline draft editing PATCH /review/drafts/{id} POST /drafts/{id}/approve Workflow → APPROVED
✦ Intelligence layer

AI that reads your vendors
so you don't have to

Every incoming WhatsApp message is parsed for intent, dates, disputes, and language — in under a second.

🧠

Multilingual intent extraction

Handles English, Gujarati, and Hindi vendor messages. Detects "moḍu" (late), "tarikh" (date), and "dekhna padega" (will check) natively.

📅

ETA variance detection

Extracts delivery dates and compares them against the original ETA. If variance exceeds 3 days, the reply is automatically staged for HITL review.

✍️

AI-drafted replies in vendor's language

When a draft is queued, an AI copywriter pre-fills an appropriate reply in Gujarati, Hindi, or English based on the detected intent and language.

🔌

Offline rule-based fallback

Works without an OpenAI API key. A deterministic keyword parser handles all major intents so the platform never goes down due to external API availability.
